## 📐 修复数学原理

### 1. 温度(softmax多样性)

```python
# 温度effect on softmax
scores = [1.0, 0.5, 0.0, -0.5]

# OLD (T=0.25): exp(x/0.25)
softmax → [0.95, 0.03, 0.01, 0.01]  ✗ 完全集中

# NEW (T=0.8): exp(x/0.8)
softmax → [0.48, 0.27, 0.16, 0.10]  ✓ 正常分布
```

**结论**: T=0.25导致只轨迹被选中，加权平均失效

### 2. 轨迹参数化

```python
# 现实轨迹数据
dx_range = [6, 14] meters (位移)

# 错误规范化
guide_acc = (dx - 20) / 12
  dx=10: (10-20)/12 = -0.833  ✗ 过负
  dx=14: (14-20)/12 = -0.500  ✗ 仍负

# 正确规范化
guide_acc = (dx - 12) / 16
  dx=6:  (6-12)/16  = -0.375  ✓ 轻微减速
  dx=12: (12-12)/16 = 0.0     ✓ 无建议(中立)
  dx=14: (14-12)/16 = 0.125   ✓ 轻微加速
```

**结论**: 参考值应在数据中点，范围应覆盖全部dx

### 3. 权重平衡

```python
# Actor Loss 权重分配
L_actor = 1.0 * L_policy 
        + 10.0 * L_cbf
        + w_traj * L_traj

# 错误配置 (w_traj=3.5)
├─ CBF约束: 10倍 (安全第一)
└─ Traj约束: 3.5倍 (接近安全)
  → 策略被两个约束夹击，最终采用保守策略

# 修复配置 (w_traj=2.0)
├─ CBF约束: 10倍 (安全第一)
└─ Traj约束: 2.0倍 (参考性)
  → SAC有更多自由度学习
```

## 💡 设计教训

### 关键洞察

1. **温度参数极其敏感**
   - T=0.7-0.8: 正常多样性
   - T=0.25: 系统崩溃
   - T>1.0: 模式分布过散

2. **参数正规化要匹配数据分布**
   - 参考值应在数据central point
   - 范围应完整覆盖数据range
   - 避免参考值在数据外

3. **权重激进升级危险**
   - 一次升级多个权重时需谨慎
   - 应分阶段:基线→微调→优化
   - 每步都要验证

4. **新增模式需充分验证**
   - 质量优于数量(4个好>5个中等)
   - 新模式需单独评估
   - 不能盲目假设更多=更好
